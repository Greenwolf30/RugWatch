How to use the website · cloud · capacity · ATC

Quick start

  1. Keep keys only in RugWatch/.env (never in the browser).
  2. Run: python run_web.py --port 8790
  3. Open http://127.0.0.1:8790/
  4. Add / upload wallets · click Push cloud to update GitHub.

Cloud (GitHub)

  • RUGWATCH_CLOUD=repo + RUGWATCH_GITHUB_REPO + GITHUB_TOKEN
  • Files: data/wallets_cloud.json (+ _002…) and data/wallets_index.json
  • 100,000 wallets per cloud file / per local DB — then auto-shard
  • ATC Render URL: raw …/data/wallets_index.json

Local vs cloud (who uses what)

  • RugWatch Scan / Monitor / Wallets pill → local DB (all shards)
  • Push cloud → GitHub index + shard files (source ATC reads on Render)
  • ATC flags → merges local + cloud when available; tags [local] [cloud] [both]
  • On Render ATC, usually cloud only via RUGWATCH_WALLETS_URL

From Actual Token Checker

  • Top nav yellow RugWatch tab opens this site
  • Analyze checkbox RugWatch includes/skips flagged wallets
  • Ruggers tab → yellow Upload on Creator / Similar / Single
  • Upload imports here and Push cloud (this site must be running)
  • Or Export JSON and use Upload manual wallets on this site
